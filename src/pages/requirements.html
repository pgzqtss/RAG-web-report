📋 Partner & Project Overview

Our client CarefulAI focuses on creating safe and reliable AI agents for various fields, including healthcare and economic development. They manage AI safety and assurance across the complete AI supply chain, from algorithm design to global management. CarefulAI hopes to benefit the society using AI, leveraging technology to make meaningful difference.

Our project, in partnership with Joseph Connor from CarefulAI, is developing a RAG system to undertake systematic reviews, serving as an important tool for NHS integration. This initiative aligns with CarefulAI's values and guides us toward creating AI applications that benefit humanity.

🎯 Project Goals

Medical Paper Analysis

Develop RAG system for comprehensive medical paper analysis

Automated Review

Generate structured systematic reviews automatically

Workload Reduction

Achieve 50%+ efficiency improvement for medical professionals

PDF Processing

Enable PDF upload & natural language query processing

👥 User Personas

Use Cases

🔧 Key Use Cases

ID Use Case Flow Result
UC01 Log in 1. User enters username and password User is logged in or sees an error message
2. System verifies login
UC02 Sign up 1. User enters registration details User is successfully registered
2. System verifies and registers the user
3. If unsuccessful, display error message
UC03 Upload PDFs 1. User selects PDF files PDFs uploaded for processing or error displayed
2. System uploads the file
3. If upload fails, error message shown
UC04 Enter Query 1. User enters query Query is sent to LLM
2. System processes the query
UC05 Generate Systematic Review 1. Retrieve data from Pinecone Review generated and available for download
2. Send query + data to OpenAI LLM
3. LLM generates review text
4. Compile structured review
UC06 Download Review 1. User requests download User receives PDF report
2. System generates PDF download
UC07 Download Graphs 1. User requests graph download User receives assessment graphs
2. System generates PDF graphs
UC08 Access History 1. User accesses history User can access historical data
2. System fetches previous records
3. User reads past reviews

Requirements Prioritization

Functional Requirements

★ Must Have
  • PDF input capability
  • Prompt-driven review definition
  • Structured review display
  • Vector database integration (RAG)
  • LLM-powered section generation
▲ Should Have
  • User authentication system
  • Review history tracking
  • Quality assessment visualization
  • PDF data cleansing
● Could Have
  • Source file visibility
  • PDF export functionality
  • Medical paper database integration
✖ Won't Have
  • Collaborative features
  • Speech interface

Non-Functional Requirements

★ Must Have
  • Web application implementation
  • RAG architecture compliance
  • Data encryption (bcrypt)
▲ Should Have
  • High performance standards
  • Maintainable codebase
● Could Have
No additional features prioritized
✖ Won't Have
  • Cross-platform compatibility
  • Multilingual support
